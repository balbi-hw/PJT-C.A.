## Learned

`model` - 프로그램이 다루는 **데이터 그 자체**로 기능보다는 데이터를 담는 역할에 집중  
  
`Repository` - 데이터를 저장하고 꺼내오는 역할. **파이프 라인**
  
`Service` - 프로그램의 **기능** 을 수행한다.
  
`Main` - 프로그램의 시작점. **Trigger**, 이 프로그램이 무엇을 하는 프로그램인지 보여주는 곳
  
> 지금까지 Main 에 만들었던 기능들 ( 메서드 ) 를 Service로 옮겨야겠다. 그리고 `Repo`도 만들어서 분류해야겠네
  
전부 분리 완료.  
  
`model` 에는 객체 정보를 담고, `repo` 는 필드를 건드리는 작업 ( 기능인데 데이터에 손이 닿는 기능을 수행 ),
`service` 에는 전반적인 기능 ( 데이터에 닿는 것 제외 ), `main` 은 프로그램의 흐름을 보여줌
  
뜬금 없지만 배구에 비유하면  
```
공 - 객체
배구 선수 - repo ( 직접 공을 만지는 사람 )
게임 규칙, 룰 - service ( 전체를 지휘하는 사람 )
관람석 - main ( 게임 밖에서 지켜보는 사람 )
```
정도로 이해했다.
